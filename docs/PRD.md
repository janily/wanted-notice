# 通缉令社区公告栏探索 Web PRD

## 1. 项目概述

本项目是一个基于 Three.js 的第一人称 WebGL 创意演示。用户进入一条老社区街道，以第一人称视角移动和观察环境，靠近社区公告栏后与一张主通缉令交互，打开高清详情页并进行缩放查看。体验以沉浸浏览为核心，加入少量线索反馈来增强“现场调查”的感觉，但不扩展为完整解谜游戏或多案件管理系统。

项目视觉和交互基线参考用户提供的示意图：左侧为街道与公告栏 3D 场景，右侧为通缉令详情面板，底部/侧边展示操作提示与项目状态信息。

## 2. 项目目标

### 2.1 产品目标

- 构建一个可运行的第一人称社区公告栏探索体验。
- 使用现有资产呈现老街道、公告栏和通缉令主题。
- 让用户能自然完成“走近公告栏、发现通缉令、查看详情、返回场景”的完整闭环。
- 通过镜头、UI、纸张、热点反馈和轻量线索提示营造沉浸感。

### 2.2 非目标

- 不制作多案件、多通缉令管理系统。
- 不制作完整解谜流程、背包系统、任务系统或剧情分支。
- 不接入后端服务，第一版数据本地静态管理。
- 不以移动端第一人称操控为主要目标，移动端只要求页面能合理展示。

## 3. 目标用户与使用场景

### 3.1 目标用户

- 想查看 Three.js WebGL 创意演示的访问者。
- 对沉浸式网页、3D 场景交互、公告栏/通缉令主题感兴趣的用户。
- 需要快速理解项目技术方案和交互架构的开发者或评审者。

### 3.2 典型场景

用户打开网页后看到老社区街道环境。通过键盘和鼠标控制视角，用户靠近公告栏，发现公告栏上的通缉令。进入可交互距离后，界面出现提示。用户按 `E` 或点击通缉令，详情面板以动画方式展开，显示完整通缉令图片。用户可滚轮缩放查看纸张细节，也可按 `Esc` 或点击关闭按钮返回 3D 场景。

## 4. 现有资源

当前项目资源位于 `assets/`：

- `assets/old_street_store.glb`：老街道/社区环境模型，约 20 MB。
- `assets/board.glb`：公告栏模型，约 5 MB。
- `assets/tjl.png`：主通缉令图片，1086 x 1448。

第一版应优先使用以上资源，不额外依赖远程图片或外部模型。

## 5. 核心体验范围

### 5.1 第一版体验定位

采用“沉浸浏览 + 少量线索点缀”的方案。

用户主要行为是探索、靠近、查看和缩放。轻线索用于增强反馈，例如首次打开通缉令时提示“已记录嫌疑人基本信息”，或用户缩放查看印章区域时提示“印章信息清晰可见”。这些反馈不构成任务进度，也不阻塞主流程。

### 5.2 核心流程

1. 用户进入页面，看到加载进度。
2. 加载完成后进入 3D 街道场景。
3. 用户使用 `WASD` 移动，鼠标控制视角。
4. 用户靠近公告栏，主通缉令出现可交互提示。
5. 用户按 `E` 或点击通缉令。
6. 镜头轻微推进或 UI 面板打开，展示 `assets/tjl.png` 的通缉令详情。
7. 用户通过滚轮缩放通缉令详情页。
8. 用户按 `Esc` 或点击关闭按钮返回场景。

## 6. 功能需求

### 6.1 场景加载

- 页面启动后显示加载界面。
- 使用 Three.js 加载 `old_street_store.glb` 和 `board.glb`。
- 使用 `assets/tjl.png` 作为主通缉令详情图，也可作为公告栏上主海报的贴图。
- 加载期间显示百分比或阶段提示。
- 加载失败时显示明确错误状态，告知用户资源未能加载。

### 6.2 3D 场景呈现

- 使用 `old_street_store.glb` 作为主要环境。
- 使用 `board.glb` 作为公告栏交互区域。
- 场景需包含基础环境光和方向光，保证公告栏与通缉令区域可辨识。
- 渲染器需支持窗口尺寸变化，画布自适应视口。
- 像素比需限制在合理范围，例如不超过 `devicePixelRatio` 与 2 的较小值，以平衡清晰度和性能。

### 6.3 第一人称控制

- 桌面端支持 `WASD` 移动。
- 鼠标控制视角。
- 进入交互或详情面板时应暂停或锁定第一人称移动，避免用户在 UI 打开时继续漂移。
- 用户移动范围需被限制在街道和公告栏附近，避免穿模、走出场景或进入不可见区域。
- 视角俯仰需限制，避免过度仰视/俯视导致迷失。

### 6.4 通缉令交互

- 公告栏上的主通缉令是唯一核心交互目标。
- 用户进入有效距离后，界面显示 `E 查看通缉令` 或等效提示。
- 鼠标指向或射线命中主通缉令时，可出现轻微高亮。
- 按 `E` 或点击命中区域后打开详情面板。
- 交互命中应使用 Three.js `Raycaster`，并只检测必要的可交互对象。

### 6.5 通缉令详情

- 详情页展示完整 `assets/tjl.png`。
- 详情页需具备关闭按钮。
- 支持 `Esc` 关闭详情页。
- 支持滚轮缩放图片，缩放范围需有限制。
- 详情页打开后，用户能清楚看到通缉令标题、照片、基本信息、特征描述、案件详情和红章。
- 缩放提示可以出现在底部，例如“滚轮缩放页面”。

### 6.6 轻线索反馈

- 首次打开通缉令详情时，显示短提示，例如“已查看通缉令详情”。
- 用户缩放到一定比例时，可显示短提示，例如“可以检查纸张细节”。
- 第一版不要求区域级图片识别；照片、案件详情、红章等区域提示属于后续扩展。
- 线索反馈只作为氛围增强，不需要进度条、任务面板或结算页面。

### 6.7 HUD 与操作说明

- 左上角展示标题，例如“中国社区 · 通缉令演示”。
- 左侧或左下角展示操作说明：
  - `W A S D` 移动
  - 鼠标控制视角
  - `E` 交互/查看详情
  - `Esc` 退出详情
  - 滚轮缩放详情页
- 底部可展示技术栈信息：
  - Three.js WebGL 项目演示
  - Three.js + TypeScript + Vite
  - GSAP 动画
- HUD 风格参考示意图，使用深色半透明面板，避免遮挡主要公告栏区域。

## 7. 技术方案

### 7.1 技术栈

- Vite
- TypeScript
- Three.js
- GSAP

### 7.2 Three.js 使用约束

参考 `threejs-fundamentals`、`threejs-loaders`、`threejs-interaction` 技能要求：

- 使用 `Scene`、`PerspectiveCamera`、`WebGLRenderer` 组织基础场景。
- 使用 `GLTFLoader` 加载 `.glb` 资源。
- 使用 `LoadingManager` 统一管理模型与图片加载进度。
- 使用 `TextureLoader` 加载 `tjl.png`。
- 贴图颜色空间应按图片贴图处理，使用适合颜色贴图的 sRGB 配置。
- 使用 `Raycaster` 处理点击和 hover 检测。
- Raycaster 只检测交互层或交互对象集合，避免全场景递归检测造成性能浪费。
- 鼠标位置计算应基于 canvas 的实际 bounds，而不是假设全窗口。
- 场景渲染循环使用 `requestAnimationFrame` 和 `Clock`，保证移动速度与帧率解耦。
- resize 时更新 camera aspect、projection matrix、renderer size。
- 销毁或重建场景时需要释放 geometry、material、texture、renderer 和事件监听器。

### 7.3 GSAP 使用约束

参考 `gsap-core`、`gsap-timeline`、`gsap-performance` 技能要求：

- 使用 GSAP 管理 UI 与镜头相关动画。
- 多步骤动画使用 `gsap.timeline()`，例如打开详情时的遮罩淡入、面板滑入、纸张缩放、提示出现。
- 优先动画 `x`、`y`、`scale`、`rotation`、`autoAlpha` 等 transform/opacity 类属性。
- 避免使用 GSAP 动画 `width`、`height`、`top`、`left` 等容易触发布局的属性。
- 对高频变化的鼠标跟随或细微热点反馈，可使用 `gsap.quickTo()`。
- 支持 `prefers-reduced-motion`，用户偏好减少动态效果时，缩短或跳过镜头推进和复杂过渡。
- 所有可反复触发的 timeline 应可复用、反向播放或在切换状态时 kill，避免残留动画。

## 8. 建议模块划分

### 8.1 `SceneManager`

职责：

- 初始化 `Scene`、`Camera`、`Renderer`。
- 管理灯光、背景、雾效或基础环境氛围。
- 管理渲染循环和窗口 resize。

依赖：

- Three.js。

### 8.2 `AssetManager`

职责：

- 使用 `LoadingManager` 加载 GLB 与 PNG。
- 暴露加载进度、成功和失败状态。
- 缓存已加载资源。

依赖：

- `GLTFLoader`
- `TextureLoader`

### 8.3 `PlayerController`

职责：

- 管理第一人称移动和视角控制。
- 处理 `WASD` 输入。
- 控制移动边界和交互状态下的暂停。

依赖：

- Three.js camera。
- DOM keyboard/mouse events。

### 8.4 `InteractionManager`

职责：

- 管理可交互对象集合。
- 处理 hover、click、`E` 键交互。
- 使用 `Raycaster` 判断是否命中主通缉令。

依赖：

- Three.js camera、renderer、raycaster。
- `NoticeSystem` 的打开详情能力。

### 8.5 `NoticeSystem`

职责：

- 管理主通缉令数据。
- 管理详情打开/关闭状态。
- 管理轻线索触发状态。

数据字段建议：

- `id`
- `title`
- `imageUrl`
- `status`
- `hasViewed`
- `hints`

### 8.6 `UIManager`

职责：

- 渲染 HUD、操作说明、加载态、错误态、详情面板和提示消息。
- 接收系统状态并同步 UI。
- 保持 UI 与 Three.js 逻辑解耦。

### 8.7 `AnimationDirector`

职责：

- 集中封装 GSAP 动画。
- 管理详情页打开/关闭 timeline。
- 管理热点高亮、HUD 显隐、轻线索提示动画。
- 根据 reduced motion 状态切换动画策略。

## 9. 状态与数据流

### 9.1 主要状态

- `loading`：资源加载中。
- `ready`：场景可交互。
- `nearNotice`：用户靠近主通缉令。
- `noticeHovered`：射线命中主通缉令。
- `noticeOpen`：通缉令详情已打开。
- `hintShown`：轻线索提示已触发。
- `error`：资源加载失败或初始化失败。

### 9.2 数据流

1. `AssetManager` 完成资源加载后通知 `SceneManager` 装配场景。
2. `PlayerController` 更新用户位置与相机方向。
3. `InteractionManager` 根据相机、鼠标和交互对象检测 hover/click。
4. 命中主通缉令后，`NoticeSystem` 切换详情状态。
5. `UIManager` 根据状态渲染提示和详情页。
6. `AnimationDirector` 根据状态切换执行 GSAP timeline。

## 10. 视觉与交互风格

- 主视觉应突出老社区、公告栏、纸质通缉令和侦查氛围。
- 场景色调以真实街道材质为主，UI 使用深色半透明样式。
- 通缉令详情保持纸张纹理清晰，不应被过度滤镜或遮罩破坏可读性。
- 面板边距、字号和缩放范围需保证桌面端阅读舒适。
- 动画应克制，强调“靠近、展开、查看”的真实感，而不是夸张动效。

## 11. 可访问性与体验约束

- 操作提示始终可见或在需要时明确出现。
- `Esc` 始终可关闭详情页。
- 加载失败时不能停留在空白画布。
- 支持 `prefers-reduced-motion`。
- 文本与控件不能互相遮挡。
- HUD 不应挡住公告栏主交互区域。

## 12. 错误处理

- GLB 加载失败：显示“模型资源加载失败，请检查 assets 文件”。
- 通缉令图片加载失败：显示“通缉令图片加载失败”，同时保留场景探索。
- WebGL 初始化失败：显示浏览器不支持或初始化失败提示。
- 进入详情时若图片未准备好：显示详情加载中状态，而不是空白面板。

## 13. 性能要求

- 首屏应显示明确加载进度。
- 鼠标 hover raycast 应节流，避免每个 mousemove 都高频递归检测复杂模型。
- 只对交互目标使用射线检测，可使用简化碰撞 mesh。
- 渲染器像素比限制为最多 2。
- 动画优先使用 transform 和 autoAlpha。
- 不创建大量同时运行的 GSAP tween。
- 场景资源维持在现有资产范围内，第一版不新增大型模型。

## 14. 验收标准

- 页面可在桌面现代浏览器中启动。
- 加载界面可显示资源加载过程。
- `old_street_store.glb` 和 `board.glb` 正常显示。
- 用户可通过 `WASD` 与鼠标探索场景。
- 用户无法明显穿模或走出主体验区域。
- 靠近公告栏后出现主通缉令交互提示。
- 用户可通过 `E` 或点击打开通缉令详情。
- 详情页展示完整 `assets/tjl.png`。
- 详情页支持滚轮缩放和 `Esc`/关闭按钮退出。
- 打开、关闭、提示和缩放反馈使用 GSAP 动画。
- reduced motion 模式下不会出现强烈镜头推进或复杂连续动画。
- 加载失败时显示错误信息。

## 15. 后续可扩展方向

第一版完成后，可以考虑以下扩展，但不纳入当前 PRD 的必须范围：

- 增加多张公告或多张通缉令。
- 增加线索收集进度。
- 增加手机端触控移动方案。
- 增加环境音效和公告栏纸张风声。
- 增加更多镜头引导或新手教程。
- 将通缉令数据抽离为 JSON 配置。
